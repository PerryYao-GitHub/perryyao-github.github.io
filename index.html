  • ArrayList 类 - 实现 List 接口

    底层数组实现, 查询效率高, 增删效率低, 线程不安全

  • LinkedList 类 - 实现 List 接口

    双链表实现, 查询效率低, 增删效率高, 线程不安全

  • Vector 类 - 实现 List 接口

    底层数组, 线程安全, 效率低 (所有方法都套上了 synchronized 关键字)

    • Stack 类 - 继承 Vector 类

  • LinkedList 类 - 实现 Queue 接口

  • PriorityQueue 类 - 实现 Queue 接口

Set 接口 - 继承 Collection 接口 - 未添加其它方法

不可重复, 无序

  • HashSet 类 - 实现 Set 接口 - 底层 HashMap

    Hash 算法 (散列算法) mod n 运算

    Set 中无 index, 所以 无 get()

    add() | remove() | contains()

    HashSet 底层是 HashMap, HashMap 底层使用 数组 和 链表 实现存储

    当两个元素 Hash 值得到位置相同时, 会 调用元素 equals() 方法, 如果相同则跳过, 不同则加到链表

    数组默认长度是 16, 数组中存储的是单链表: NodeList<Object>[] hashArr = new NodeList<>[16];

    使用 HashSet 存储自定义对象

  • TreeSet 类 - 实现 Set 接口 - 底层 TreeMap

    允许对元素排序 要给定排序规则

    通过元素自身实现比较规则 (Integer, String) compareTo() 方法

    通过比较器指定比较规则

Knapsack (0-1) 背包

原型

给出total种选择, 每种选择有对应的cost和value, 在cost不超过capacity时, 最大化value的总和

二维 dp

  • dp[i][c]: 从 0, 1, ..., i 选择中选取, 其对应的总 cost 不超过 c 的条件下, 能达到的总 value 最大值
  • 两个 for 可交换顺序, 但最好先 i 后 c
  • calculate 时, i 从 1 开始, c 从 0 或 1 开始都行

一维 dp (滚动数组)

  • dp[c]: 从 0, 1, ..., total - 1 选择 (全部选择) 中选取, 在总 cost 不超过 c 的条件下, 总 value 的最大值
  • 必须倒序
  • for 不能颠倒

变形1: 找最接近某个值的组合

给出一个非负数组nums和目标值target, 找出nums中, 和最接近target的值

相当于原型中cost[i] == value[i]

二维 dp

  • dp[i][t]: 选取 0, 1, ..., i 之间的数字, 其和能够达到 target (不能超过) 的最接近的值

一维 dp

  • dp[t]: 选取 0, 1, ..., total - 1 之间的数字 (全部数字), 其和能够达到 target (不能超过) 的最接近的值

变形2: 找和等于某个值有多少种方法

给一个非负数组nums和一个target, 寻找从nums中选取一些数, 使其和等于target一共有多少种方法

二维 dp

  • dp[i][t]: nums 从 0, 1, ..., i 中选择, 其和等于 t 的方法数

一维 dp

  • dp[t]: 可选所有 nums, 其和 == t 的方法数

变形3: 二维背包

同样是 n 种物品, 对应 value 数组, 但物品的约束有两个, (例如 重量 空间)

有 cost0 数组, 上限是 C0 (capacity0)

有 cost1 数组, 上限是 C1 (capacity1)

三维 dp

  • c0, c1 从 0 开始的原因: 这里对于 init i 的操作是将(i, 0, 0) i = 0, …, total - 1 一整条线初始化成 0, 而不是 i - c0 和 i - c1 两个平面
