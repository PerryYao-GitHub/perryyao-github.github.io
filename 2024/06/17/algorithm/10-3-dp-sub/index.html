DP - Subarray & Subsequence

Posted on 2024-06-17 Edited on 2025-02-02 In algorithm

子序列类型

Subarray - 连续子序列

Subsequence - 非连续但是保序的子序列

Subset - 不连续不保序 (集合本无顺序)

两类 dp 的定义

定义结尾 (固定结尾) - dp 本身不包含问题的答案, 而是在外部使用一个res接收dp数组中的最大值作为答案

  • 以 i结尾 的最长递增 Subsequence 的长度
  • 以 i结尾 的最长递增 Subarray 的长度

定义范围 (不固定结尾) - dp 本身包含问题答案, dp 数组的最后一项就是答案

  • text1长度为i的子串和text2长度为j的子串中, Longest Common Subsequence

定义几个抽象概念 (函数)

生成子序列函数

结合取范围记号, 把每一个题目的 dp 含义给写清楚

Overview

最关键的问题莫过于 dp 数组定义的确定.

当我们把 dp 的定义设置为仅与 i[0..len] 相关, 或仅与 subarrs(it, endwith=false) or subseqs(it, endwith=false) 时, 就相当于是在解原问题的子问题 — 先解决小规模的问题, 再把规模扩大, 于是 dp 数组最后的一项就是原问题的答案.

  • dp[i][j]: 集合 subseqs(nums1[0..i], endwith=false) 和集合 subseqs(nums2[0..j], endwith=false) 中, 找相等的元素, 取最长的那一组的长度
    • 1143. Longest Common Subsequence
    • 1035. Uncrossed Lines
    • 392. Is Subsequence
  • dp[i][j]: 集合 subseqs(s[0..i], endwith=false) 中, 与 t[0..j] 相等元素的个数
    • 115. Distinct Subsequences
  • dp[i][j]: word1[0..i] 和 word2[0..j] 需要经过删除后相等的最小次数
    • 583. Delete Operation for Two Strings
  • dp[i][j]: word1[0..i] 和 word2[0..j] 需要编辑的最小次数
    • 72. Edit Distance

当我们把 dp 的定义设置为与 subarrs(it, endwith=true) or subseqs(it, endwith=true) 时, 子问题不是原问题的缩小, 原问题不是子问题的扩大, 这时的答案与 dp 数组的最后一位没有关系, 答案往往是 dp 数组的最值.

  • dp[i]: 集合 subseqs(nums[0..=i], true) 中, 递增元素的最大长度
    • 300. Longest Increasing Subsequence
  • dp[i]: 集合 subarrs(nums[0..=i], true) 中寻找递增元素的最大长度
    • 674. Longest Continuous Increasing Subsequence
  • dp[i]: 集合 subarrs(nums[0..=i], true) 中, 选择和最大的元素的和
    • 53. Maximum Subarray
  • dp[i][j]: 集合 subarrs(nums1:[0..i], endwith=true) 和集合 subarrs(nums2[0..j], endwith=true) 中, 找相同的元素, 取最长的那一组的长度 (最大公共后缀长度)
    • 718. Maximum Length of Repeated Subarray

我们发现, 如果是关于 Subarray 的问题, 就要设 endwith = true (Subarrs(it, endwith = true)); 而关于 Subsequence 的问题, 则不一定.

5. Longest Palindromic Substring

暴力法

  • 时间复杂度: O(n^3)

  • 空间复杂度: O(1)

中心扩展法

  • 思路： 以字符串的每个字符和相邻两个字符作为中心，向两边扩展，直到遇到不相等的字符。
  • 优点： 相比暴力解法，减少了不必要的判断。
  • 时间复杂度： O(n^2)，比暴力解法有所优化。
